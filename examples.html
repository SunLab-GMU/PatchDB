Typical Examples

Security Patches contains multiple categories. In PatchDB, apart from the non-security patches, we further manually classify the security patches into serveral sub-classes, including

  • buffer overflow
  • improper input validation
  • info leakage
  • double free/use after free
  • integer overflow
  • NULL pointer dereference
  • improper authentication
  • uncontrolled resource consumption
  • race condition
  • unintialized use
  • et al.

Below are some typical examples in PatchDB.

Date: Wed, 27 Aug 2014 14:42:54 -0700
Subject: [PATCH] KVM: x86: Improve thread safety in pit

There's a race condition in the PIT emulation code in KVM.  In
__kvm_migrate_pit_timer the pit_timer object is accessed without
synchronization.  If the race condition occurs at the wrong time this
can crash the host kernel.

This fixes CVE-2014-3611.
